# Xca-Tutor 头脑风暴会议纪要

**会议时间：** 2026-04-07  
**参会角色：**
- **产品经理 (PM)**：提出需求、权衡取舍
- **技术架构师 (Architect)**：评估可行性、设计系统
- **用户体验设计师 (UX)**：关注交互流程
- **秘书 (Recorder)**：记录过程、整理结论

**会议目标：** 定义 Xca-Tutor 的产品形态和技术方案

---

## 第一轮：产品形态

**PM：** 用户想要一个英语陪练 Agent，核心需求：
1. 口语练习
2. 听力训练
3. 词汇拓展

**UX：** 关键是语音交互。但纯语音有个问题：用户怎么知道自己在进步？怎么回顾学的内容？

**Architect：** 技术上需要：STT（语音转文字）+ LLM（对话大脑）+ TTS（文字转语音）。本地模型不行，得用云端。

**PM：** 用户提出一个结合方案：纯语音对话，但必须详细记录，支持回看。还要场景化训练。

**结论：**
- ✅ 形态 = 纯语音对话 + 详细记录 + 场景训练
- ✅ 云端方案（OpenAI Whisper + GPT-4o + TTS）

---

## 第二轮：使用场景与难度适配

**UX：** 用户说"全场景"——通勤、居家、办公都要支持。这意味着：
- 通勤：可能不能看屏幕，纯音频
- 居家/办公：可以看屏幕，查看记录和复盘

**PM：** 难度适配用户要求"自动调整但显性展示"。不像有些 App 暗中调整，用户完全不知道。

**Architect：** 可以在界面加状态栏，实时显示当前难度等级和自适应依据。

**结论：**
- ✅ 全场景支持，界面根据场景自适应
- ✅ 难度自动调整，但过程可见（状态栏展示）

---

## 第三轮：纠错与复盘机制

**UX：** 纠错方式是个关键体验点。用户选"说完再纠"——这符合语言习得理论，不打断流畅度。

**PM：** 但纠完了就结束吗？用户明确要求：
- 段落报告（总结本轮错误）
- 重复训练方法（针对错误进行重练）
- 主动提前完成（练了几遍觉得会了，可以跳过）

**Architect：** 这需要把错误结构化存储，生成"错题本"，支持按错误类型筛选重练。

**结论：**
- ✅ 纠错方式：说完再纠
- ✅ 段落报告：对话结束后生成
- ✅ 重复训练：错题本 + 自由重练 + 强制复习

---

## 第四轮：场景训练设计

**PM：** 用户选择 B+C 结合：开放沙盒 + 任务制。

**UX：** 我理解是：进入场景后可以自由对话，但系统有隐藏任务。完成任务有评分，但不限制自由发挥。

**例子（餐厅场景）：**
- 隐藏任务：点餐 + 要求特殊口味 + 询问推荐
- 用户自由发挥：可以直接投诉，服务员（Agent）会应对
- 最后评分：任务完成度 + 语法分 + 流利度

**结论：**
- ✅ 场景训练 = 开放沙盒 + 隐藏任务
- ✅ 多维度评分（任务/语法/流利度）

---

## 第五轮：记录与复盘

**PM：** 用户要 A+B+C 全部：实时字幕 + 课后报告 + 时间轴回放。

**UX：** 这相当于一个完整的"播放器"体验：
- 实时：边聊边看字幕
- 课后：报告总结
- 回放：像听歌一样拖动进度，重听某段对话

**Architect：** 技术实现：音频分段存储 + 时间戳对齐文字。回放时根据时间戳定位音频位置。

**结论：**
- ✅ 实时字幕流
- ✅ 课后报告（语法/词汇/流利度）
- ✅ 时间轴回放（可拖动定位）

---

## 第六轮：技术选型与配置需求

**Architect：** 用户最终选择：
- 语音：OpenAI Whisper + TTS（云端，质量优先）
- 平台：先做 macOS，验证后再扩其他平台
- 存储：SQLite 本地存储
- 名称：Xca-Tutor

**PM：** 用户新增一个重要需求：模型参数需要可以在界面中手动配置。这意味着不能写死在代码里。

**UX：** 这很好，用户可以根据自己的需求调整模型行为。比如：
- 想对话更流畅，可以降低 temperature
- 想模型更有创意，可以调高 temperature
- 可以切换不同的模型版本（GPT-4o / GPT-4o-mini）

**Architect：** 需要设计一个设置页面，支持配置：
- API Key
- 模型选择（GPT-4o, GPT-4o-mini, GPT-4-turbo 等）
- Temperature（创造性/稳定性）
- Max Tokens（回复长度限制）
- 语音音色选择

**结论：**
- ✅ 技术栈：Swift + SwiftUI + OpenAI API + SQLite
- ✅ 平台策略：macOS MVP → iOS → Windows → Android → 鸿蒙
- ✅ 模型参数支持界面手动配置

## 会议结论

### 产品定义
- **名称：** Xca-Tutor
- **定位：** 私人英语陪练 Agent
- **核心体验：** 场景化语音对话 + 智能复盘 + 针对性重练

### MVP 功能（macOS）
1. 语音对话核心（Whisper + GPT-4o + TTS）
2. 5 个预设场景（餐厅、机场、面试、酒店、购物）
3. 实时字幕 + 时间轴回放
4. 段落报告（语法/词汇/流利度）
5. 错题本 + 重复训练
6. 难度自适应（显性展示）

### 技术方案
- **平台：** macOS（Swift + SwiftUI）
- **语音：** OpenAI API
- **数据：** SQLite
