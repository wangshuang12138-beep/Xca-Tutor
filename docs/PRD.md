## 1. 产品概述

### 1.1 什么是 Xca-Tutor？

Xca-Tutor 是一个专注于英语口语、听力、词汇训练的 AI 陪练工具。它通过语音交互，让用户在真实场景中练习英语，并提供详细的复盘报告和针对性重练。

**一句话描述：** 像有一个 24 小时在线的英语私教，陪你练口语、纠正错误、帮你复盘。

### 1.2 为什么做这个项目？

**现有方案的问题：**

| 方案 | 问题 |
|------|------|
| 真人外教 | 贵（¥200-500/小时）、预约麻烦、不敢犯错 |
| 英语学习 App | 机械跟读、不自然、缺乏真实对话感 |
| ChatGPT 语音 | 没有结构化训练、不会主动纠错、没有复盘 |

**Xca-Tutor 的差异化：**
- **场景化训练** —— 不只是闲聊，在特定场景（餐厅、机场、面试）中练习
- **智能复盘** —— 对话后生成详细报告，知道哪里错了、怎么改进
- **针对性重练** —— 错题本功能，重点攻克薄弱环节
- **难度自适应** —— 根据你的水平自动调整，始终处于"够得着"的挑战区

### 1.3 目标用户

1. **职场人士** —— 需要商务英语、面试准备
2. **留学生/准留学生** —— 适应国外生活场景
3. **英语爱好者** —— 保持语感、拓展词汇
4. **备考人群** —— 雅思、托福口语练习

### 2.2 功能详解

#### 2.2.1 场景训练系统

**场景定义：**
每个场景是一个"沙盒+任务"的组合：
- **沙盒**：可以自由对话，不限定台词
- **隐藏任务**：系统预设的目标，完成后有奖励

**MVP 场景（5个）：**

| 场景 | 角色设定 | 隐藏任务示例 |
|------|----------|-------------|
| 餐厅点餐 | 你是顾客，Agent 是服务员 | 点餐、要求特殊口味、询问推荐、投诉 |
| 机场登机 | 你是旅客，Agent 是地勤 | 询问登机口、行李问题、升舱咨询 |
| 求职面试 | 你是候选人，Agent 是面试官 | 自我介绍、回答行为问题、反问环节 |
| 酒店入住 | 你是住客，Agent 是前台 | 办理入住、要求换房、咨询设施 |
| 购物 | 你是顾客，Agent 是店员 | 询问商品、砍价、退换货 |

**场景流程：**

```
选择场景
    │
    ▼
[难度评估] —— 根据历史数据或简短测试
    │
    ▼
进入场景 —— 显示背景信息（你在一家意大利餐厅）
    │
    ▼
对话开始 —— Agent 主动打招呼
    │
    ▼
自由对话 —— 你说什么 Agent 都能接
    │
    ▼
完成任务 —— 达成隐藏目标或主动结束
    │
    ▼
生成报告 —— 多维度评分 + 错误分析
```

#### 2.2.2 语音交互核心

**技术链路：**

```
用户语音
    │
    ▼
[麦克风采集]
    │
    ▼
[OpenAI Whisper API] —— 语音识别
    │
    ▼
[文本 + 上下文] → [OpenAI GPT-4o] —— 对话大脑
    │
    ▼
[回复文本] → [OpenAI TTS API] —— 语音合成
    │
    ▼
[扬声器播放]
```

**功能特性：**

| 特性 | 说明 |
|------|------|
| 实时字幕 | 边说边显示文字，延迟 < 500ms |
| 打断支持 | 可以随时打断 Agent 说话 |
| 语速控制 | 可以要求 Agent "说慢点" |
| 重复播放 | 可以要求 Agent "再说一遍" |

#### 2.2.3 记录与复盘系统

**实时记录：**
- 音频分段存储（每段 5-10 秒）
- 时间戳对齐文字
- 显示说话者标识（你 / Agent）

**时间轴回放：**

```
[━━━━━●━━━━━━━━━━━━━━━━━━━━━━━]  12:34
  ↑
拖动定位，点击播放该段对话
显示对应文字 + 说话者 + 时间戳
```

**段落报告（对话结束后）：**

```markdown
# 练习报告：餐厅点餐

## 总体评分：B1（中级）
- 任务完成度：80%（4/5 个目标达成）
- 语法准确度：85%（3 处小错误）
- 流利度：78%（有 2 处明显停顿）

## 错误分析

### 语法错误（2 处）
1. "I want a steak medium rare" → "I'd like a steak cooked medium rare"
   - 问题：want 太直接，cooked 遗漏
   - 建议：点餐用 "I'd like" 更礼貌

2. "No vegetables" → "Without vegetables, please"
   - 问题：直接否定不够礼貌
   - 建议：用 "without" 更自然

### 词汇亮点（3 处）
- ✅ 使用了 "recommend"（推荐）
- ✅ 使用了 "specialty"（招牌菜）
- ✅ 使用了 "check, please"（买单）

### 发音提示
- "vegetables" 重音在第一个音节 /ˈvɛdʒtəbəlz/

## 建议重练
- [ ] 礼貌点餐句式（3 个变体）
- [ ] 投诉/退换货表达

[开始重练] [保存到错题本] [分享]
```

#### 2.2.4 重复训练系统

**错题本：**

```
┌─────────────────────────────────────┐
│ 我的错题本                    共 23 条 │
├─────────────────────────────────────┤
│ 🔴 语法错误（8）                     │
│    I'd like vs I want...            │
│                                     │
│ 🟡 词汇盲区（10）                    │
│    餐厅相关词汇...                   │
│                                     │
│ 🟢 发音问题（5）                     │
│    vegetables...                    │
└─────────────────────────────────────┘
```

**重练模式：**

1. **智能推荐**
   - 系统分析哪些错误你容易再犯
   - 优先让你练高频错误

2. **自由重练**
   - 错题本中自主选择
   - 标记"已掌握"跳过

3. **强制复习**
   - 重要错误必须练 1-3 遍
   - 可以提前完成

**重练流程：**

```
选择错误条目
    │
    ▼
[显示原对话上下文]
    │
    ▼
[针对性练习] —— 系统给出提示，让你重新说
    │
    ▼
[AI 评估] —— 说对了通过，错了继续
    │
    ▼
[标记状态] —— 已掌握 / 需再练
```

#### 2.2.5 难度自适应系统

**难度等级：**

| 等级 | CEFR | 描述 |
|------|------|------|
| A1 | 入门级 | 简单句，基础词汇 |
| A2 | 初级 | 日常话题，简单交流 |
| B1 | 中级 | 工作/学习，描述经历 |
| B2 | 中高级 | 复杂话题，流利表达 |
| C1 | 高级 | 专业话题，精准用词 |
| C2 | 精通级 | 接近母语水平 |

**自适应策略：**

```
初始评估
    │
    ▼
[简短测试] —— 5 个问题，判断基础水平
    │
    ▼
设定初始难度（如 B1）
    │
    ▼
对话中实时监测
    ├── 流利度 > 90% + 无错误 → 考虑升级
    ├── 流利度 70-90% + 少量错误 → 保持
    └── 流利度 < 70% + 多错误 → 考虑降级
    │
    ▼
显性提示："检测到你的水平提升，是否升级难度？"
```

```

│ 🎯 当前难度：B1（中级）                  │
│ 📈 流利度：85%  │  🎯 准确度：82%       │
│ 💡 建议：可以尝试更复杂的句式            │
```

## 5. 技术需求

### 5.1 API 依赖

| 服务 | 用途 | 预估成本 |
|------|------|----------|
| OpenAI Whisper | 语音识别 | ~¥1.8/天 |
| OpenAI GPT-4o | 对话大脑 | ~¥0.1/天 |
| OpenAI TTS | 语音合成 | ~¥0.01/天 |
| **合计** | | **~¥2/天（30分钟练习）** |

### 5.2 数据存储

```sql
-- 用户配置表
CREATE TABLE user_settings (
    id INTEGER PRIMARY KEY,
    api_key TEXT,
    use_proxy BOOLEAN DEFAULT FALSE,
    -- 模型参数配置
    chat_model TEXT DEFAULT 'gpt-4o',
    temperature REAL DEFAULT 0.7,
    max_tokens INTEGER DEFAULT 2000,
    voice_name TEXT DEFAULT 'alloy',
    whisper_model TEXT DEFAULT 'whisper-1',
    -- 练习偏好
    default_difficulty TEXT DEFAULT 'B1',
    auto_upgrade BOOLEAN DEFAULT TRUE,
    correction_strictness TEXT DEFAULT 'standard',
    -- UI 设置
    theme TEXT DEFAULT 'auto',
    language TEXT DEFAULT 'zh-CN'
);

-- 对话记录表
CREATE TABLE conversations (
    id TEXT PRIMARY KEY,
    scene TEXT,
    start_time INTEGER,
    end_time INTEGER,
    difficulty TEXT,
    audio_path TEXT
);

-- 消息表
CREATE TABLE messages (
    id TEXT PRIMARY KEY,
    conversation_id TEXT,
    role TEXT, -- 'user' | 'agent'
    content TEXT,
    audio_segment INTEGER,
    timestamp INTEGER,
    FOREIGN KEY (conversation_id) REFERENCES conversations(id)
);

-- 错误记录表
CREATE TABLE mistakes (
    id TEXT PRIMARY KEY,
    conversation_id TEXT,
    type TEXT, -- 'grammar' | 'vocabulary' | 'pronunciation'
    original TEXT,
    correction TEXT,
    explanation TEXT,
    mastered BOOLEAN DEFAULT FALSE,
    practice_count INTEGER DEFAULT 0
);
```

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 目标 | 说明 |
|------|------|------|
| 语音延迟 | < 1s | 用户说完到 Agent 开始回应 |
| 字幕延迟 | < 500ms | 实时字幕显示 |
| 启动时间 | < 3s | 冷启动到可交互 |
| 报告生成 | < 5s | 对话结束到报告生成 |

### 6.2 安全与隐私

- API Key 本地加密存储（Keychain）
- 音频文件本地存储，不上传（除非用户选择分享）

### 8.1 付费模式
**方案：** 用户自备 OpenAI API Key

### 8.2 场景内容来源
**方案：** A + C（自制 + 用户自定义）

- 场景格式开放，高级用户可自行创建自定义场景
- 场景配置文件使用 JSON 格式，便于扩展

### 8.3 发音评估
**方案：** A（第一版不做）
- MVP 专注于语法、词汇、流利度评估
- 发音评估延后到 v0.3 版本（需要接入专门的发音评估算法）
